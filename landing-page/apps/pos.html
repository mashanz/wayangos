WayangPOS

Touchscreen Point of Sale for Small Businesses

Direct framebuffer rendering. SQLite database. Runs on any hardware with HDMI output.

⬇ Download ISO View Source →

WayangPOS running on WayangOS — direct framebuffer rendering, zero dependencies

Features

Everything a small business POS needs. Nothing it doesn't.

🖥️

Direct Framebuffer

No X11, no Wayland, no SDL2. Renders directly to /dev/fb0 for maximum performance on minimal hardware.

🔐

User Management

Admin and cashier accounts with password login. CRUD user management.

📋

Menu Management

Add, edit, delete menu items with categories (Food/Drinks/Snacks). Set prices and manage inventory.

💰

Order & Payment

Cash and QRIS payment methods. Order history with timestamps.

⌨️

Virtual Keyboard

Full QWERTY on-screen keyboard for touchscreen-only setups. Works without physical keyboard.

🏪

Store Settings

Customizable store name, admin-configurable via settings page.

📊

Order History

Browse past orders with pagination. Track sales over time.

🔧

Minimal Hardware

Runs on Orange Pi Zero 2W (~$15), Raspberry Pi, or any x86 PC. Total hardware cost under $100.

Hardware

Run a full POS system for under $100.

Requirements

  • Any x86_64 PC or ARM64 SBC with HDMI output
  • Minimum 64MB RAM
  • USB keyboard (touchscreen optional)
  • Recommended: Orange Pi Zero 2W + HDMI touchscreen + USB thermal printer

Build from source:

# Clone the repo
git clone https://github.com/mashanz/wayangos-pos.git
cd wayangos-pos

# Build (requires musl-gcc + SQLite amalgamation)
make

# Build bootable ISO
make iso

Tech Specs

For developers who want to know what's under the hood.

Language C (single file, ~2700 lines)
Rendering Direct framebuffer (/dev/fb0) via mmap
Database SQLite3 (amalgamation, statically linked)
Font VGA 8×16 bitmap (embedded in binary)
Input evdev (keyboard + mouse + touchscreen)
Binary Size ~2.2MB (statically linked, musl + SQLite)
Rendering Double buffered (no flicker)
